#### Fixed concurrent write conflict in `terragrunt stack generate`

On nested stack hierarchies, `terragrunt stack generate` could produce non-deterministic `file exists`, `no such file or directory`, or `unexpected EOF` errors under parallel use. Two bugs caused this: the dedup map treated string-aliased paths as distinct, and in-process concurrent calls raced on the shared output tree. Stack-file paths are now cleaned and absolutized at discovery, and in-process calls serialize on a mutex. As a result, `ListStackFiles` and `ListStackFilesWithExcludes` now return absolute paths.
