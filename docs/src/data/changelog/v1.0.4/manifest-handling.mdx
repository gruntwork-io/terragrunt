#### Hardened module manifest cleanup

Terragrunt tracks files copied into a module's working directory using `.terragrunt-module-manifest` so subsequent runs can clean up files that have been removed from the source. The cleanup pass now anchors all reads and removals against the manifest folder via `os.Root`, and any decoded entry that resolves outside that folder is dropped with a warning instead of acted on.

Any pre-existing `.terragrunt-module-manifest` files inside a fresh source download are also cleared before cleanup runs, so a manifest shipped with downloaded module content cannot influence what cleanup deletes.
