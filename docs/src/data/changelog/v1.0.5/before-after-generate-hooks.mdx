#### `before_hook` and `after_hook` now support the `generate` command

You can now run hooks immediately before or after Terragrunt evaluates a unit's `generate` blocks
(and the `remote_state.generate` attribute, when configured) by setting `commands = ["generate"]`
on a hook. This fills the gap between `init-from-module`, which only fires when source is
downloaded, and `init`, which runs after generation. The hook working directory is the cache
working directory, so relative paths in the `execute` command reference files alongside the
generated output.

```hcl
terraform {
  before_hook "cleanup" {
    commands = ["generate"]
    execute  = ["bash", "-c", "rm -f tgen_*"]
  }

  after_hook "stamp" {
    commands = ["generate"]
    execute  = ["bash", "-c", "echo $(date -u) > .last_generated"]
  }
}
```
