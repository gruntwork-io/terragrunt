#### `stack output` fetches unit outputs in parallel

`terragrunt stack output` previously read outputs from each unit sequentially, which became slow on large stacks because every read blocks on a `tofu output` child process. Outputs are now fetched concurrently across units, bounded by the `--parallelism` flag (default: number of CPU cores; env `TG_PARALLELISM`).

```bash
terragrunt stack output --parallelism 4
```

Raise `--parallelism` to push throughput above the CPU-core default, or lower it on machines where higher concurrency would exhaust file descriptors or hit cloud-provider rate limits.
