#### `overwrite-terragrunt-or-skip` experiment adds a new `if_exists` mode for `generate` blocks

Enable the new [`overwrite-terragrunt-or-skip`](/reference/experiments/active#overwrite-terragrunt-or-skip) experiment to use `if_exists = "overwrite_terragrunt_or_skip"` in `generate` blocks.

When the target file was previously generated by Terragrunt, it is overwritten as usual. When the file was not generated by Terragrunt, generation is skipped and the file is left unchanged. No error is returned in either case.

This is useful when a hand-written file sits alongside Terragrunt-managed files and you want Terragrunt to leave it alone rather than fail:

```hcl
generate "backend" {
  path      = "backend.tf"
  if_exists = "overwrite_terragrunt_or_skip"
  contents  = <<EOF
terraform {
  backend "s3" {
    bucket = "my-state-bucket"
    key    = "terraform.tfstate"
    region = "us-east-1"
  }
}
EOF
}
```

Using this value without enabling the experiment returns an error.
