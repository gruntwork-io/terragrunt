#### `stack-dependencies`: `autoinclude` deep-merges like a regular include

A generated `terragrunt.autoinclude.hcl` now merges into the unit or stack using the same `deep_merge` semantics as a regular `include`, applied uniformly across both generation and run. The `inputs` map is deep-merged so nested keys from the unit and the autoinclude both survive, while `generate` and `remote_state` are whole-block overrides, matching how includes behave. The autoinclude is the winning side on conflicts, and `locals` stay local in scope.
