#### `stack-dependencies`: autoinclude resolves stack-level references at generate time

An `autoinclude` block is evaluated in the context of the `terragrunt.stack.hcl` file where it is declared. Every expression is resolved at `terragrunt stack generate` time, so the generated `terragrunt.autoinclude.hcl` contains rendered literals: stack-level `local.*`, `unit.<name>.path`, `stack.<name>.path`, and terragrunt/OpenTofu function calls (such as `get_repo_root`, `get_terragrunt_dir`, `read_terragrunt_config`, `run_cmd`) all resolve against the stack file. The only deferred reference is `dependency.<name>.outputs.<key>`: any expression that references it is kept verbatim and resolved inside the generated unit at run time, including inside the `inputs` block.

Two patterns are now rejected at generate time with a clear error: a `values.*` reference anywhere in the autoinclude body (values is a unit-scoped namespace that is not available in the stack file; use a stack-level local declared in `terragrunt.stack.hcl` or a literal), and a `locals` block defined inside the autoinclude (declare locals at the stack level instead). A dependency `config_path` may still reference `values.*` for the cross-level path-passing pattern, since `config_path` is fully resolved at generate time.
