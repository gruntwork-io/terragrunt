#### `stack-dependencies`: resolve generate-time references in autoinclude `mock_outputs`

A `local.*` (or `unit.<name>.path` / `stack.<name>.path`) reference inside an `autoinclude` dependency's `mock_outputs` is now resolved at `terragrunt stack generate` time, the same as the `values` block. Previously such references were written to the generated `terragrunt.autoinclude.hcl` verbatim instead of being substituted with their value, which was inconsistent and surprising.

References that are only known later are kept verbatim and resolved when the generated unit is parsed: `values.*` (the unit's own values block) and `dependency.<name>.outputs.<key>`. This matches how the autoinclude `inputs` block already behaves.
