#### `stack-dependencies`: autoinclude resolves stack-level references at generate time

An `autoinclude` block is evaluated in the context of the `terragrunt.stack.hcl` file where it is declared. Stack-level `local.*`, `unit.<name>.path`, `stack.<name>.path`, functions, and literals resolve to their values at `terragrunt stack generate` time, including inside the `inputs` block. Only `dependency.<name>.outputs.<key>` is kept verbatim and resolved inside the generated unit at run time.

Two references are now rejected at generate time with a clear error: a `values.*` reference anywhere in the autoinclude body (values is a unit-scoped namespace that is not available in the stack file; use a stack-level local declared in `terragrunt.stack.hcl` or a literal), and a `locals` block defined inside the autoinclude (declare locals at the stack level instead). A dependency `config_path` may still reference `values.*` for the cross-level path-passing pattern, since `config_path` is fully resolved at generate time.
