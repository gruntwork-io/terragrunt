#### `stack output` works without `terragrunt.stack.hcl` (implicit stacks)

Gated behind the new [`stack-output-implicit`](/reference/experiments/active/#stack-output-implicit---what-it-does) experiment.

`terragrunt stack output` previously required a `terragrunt.stack.hcl` file to produce aggregated outputs. When run in a directory with only loose units, it warned that no stack files were found and produced nothing.

With the experiment enabled, it now discovers units that are not declared in any stack and aggregates their outputs keyed by the unit's directory path (relative to the working directory, forward slashes on every OS). In a tree that also contains explicit stacks, each stack's output tree is nested under the stack directory's relative path, so every key reflects a filesystem position and explicit and implicit entries cannot collide (a stack file in the working directory itself gets no prefix):

```bash
$ terragrunt stack output
db = {
  endpoint = "db.local"
  port     = 5432
}
vpc = {
  vpc_id = "vpc-1234567890"
}
apps/api = {
  domain = "api.example.com"
}
```

Filtering and drill-down work the same way they do for explicit stacks:

```bash
$ terragrunt stack output vpc.vpc_id --format raw --experiment stack-output-implicit
vpc-1234567890
```

Directory names containing `.` cannot be drilled into with the dot separator — use `--format json` for those, same caveat as explicit-stack unit names with `.`.

Closes [#4559](https://github.com/gruntwork-io/terragrunt/issues/4559).
