#### Fix permission denied when a `generate` block overwrites a CAS-materialized file

`generate` blocks with `if_exists = "overwrite"` no longer fail with `permission denied` when the target
file already exists in a CAS-materialized module source (for example, a module that ships its own
`versions.tf`). The read-only file is replaced with a writable one, and the shared CAS store is never modified.
