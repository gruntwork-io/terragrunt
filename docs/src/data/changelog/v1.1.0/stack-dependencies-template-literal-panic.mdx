#### `stack-dependencies`: fix panic on a non-string literal interpolation in an autoinclude template

`terragrunt stack generate` no longer panics when an `autoinclude` template interpolates a non-string literal
(for example `"${0}"` or `"${true}"`) alongside a `dependency.*` reference. The interpolated literal is now
rendered to its string form (`${0}` becomes `0`) and the dependency reference is preserved for the unit.
