#### Filtering a stack that calls another stack now generates and runs the whole subtree

When a stack includes another stack, filtering the parent used to stop one level early. `terragrunt stack generate --filter './my-stack | type=stack'` only generated the first level, and `terragrunt stack run --filter './my-stack | type=stack' -- apply` reported "No units discovered".

Filtering a stack now generates and runs its whole subtree, so `terragrunt stack run --filter './my-stack | type=stack' -- apply` applies every unit inside `my-stack`.
