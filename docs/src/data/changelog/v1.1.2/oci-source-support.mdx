#### OCI registry support for module sources, stack units, and stack CAS processing

Terragrunt now supports `oci://` as a source scheme for fetching OpenTofu/Terraform modules, stack `unit {}` sources, and stack `stack {}` sources from any OCI-compliant registry (e.g. GitHub Container Registry, Amazon ECR, Docker Hub, self-hosted registries).

**Supported source locations:**

- `terraform { source }` in unit `terragrunt.hcl` files
- `unit { source }` blocks in `terragrunt.stack.hcl`
- `stack { source }` blocks in `terragrunt.stack.hcl`
- CAS (`update_source_with_cas = true`) stack processing

**URL format:**

```
# Standard OCI image reference notation
oci://<registry>/<repository>:<tag>[//<subdir>]
oci://<registry>/<repository>@<digest>[//<subdir>]

# OpenTofu-compatible query parameter notation
oci://<registry>/<repository>[//<subdir>]?tag=<tag>
oci://<registry>/<repository>[//<subdir>]?digest=<digest>
```

Examples:

```hcl
# Unit source — OpenTofu module packaging spec (zip archive)
terraform {
  source = "oci://ghcr.io/my-org/terraform-modules?tag=v1.2.3"
}

# Standard image reference notation
terraform {
  source = "oci://ghcr.io/my-org/terraform-modules:v1.2.3"
}

# With subdir selector
terraform {
  source = "oci://ghcr.io/my-org/terraform-modules:v1.2.3//modules/vpc"
}

# Stack unit block
unit {
  source = "oci://registry.example.com/infra/vpc-module:v2.0.0"
  path   = "vpc"
}
```

**Supported archive formats:**

| Layer media type | Format | Notes |
|---|---|---|
| `archive/zip` | ZIP | OpenTofu module packaging spec (preferred) |
| `application/vnd.oci.image.layer.v1.tar+gzip` | tar.gz | Standard OCI image layer |
| `application/vnd.terraform.module.v1+tar` | tar.gz | Legacy Terraform module format |
| `application/octet-stream` | tar or tar.gz | Auto-detected by magic bytes |

**Authentication:**

Credentials are resolved in priority order:

1. `TG_OCI_TOKEN` environment variable (Bearer token)
2. `TG_OCI_USERNAME` + `TG_OCI_PASSWORD` environment variables (Basic auth)
3. Docker credential store (`~/.docker/config.json`)
4. Unauthenticated (public registries)

**Pushing modules to an OCI registry:**

Use [ORAS](https://oras.land/) to package and push modules. For compatibility with OpenTofu's native OCI module support, use the OpenTofu module packaging spec (zip + `application/vnd.opentofu.modulepkg` artifact type):

```bash
# OpenTofu module packaging spec (zip archive)
zip -r module.zip /path/to/module
oras push ghcr.io/my-org/terraform-modules:v1.2.3 \
  --artifact-type application/vnd.opentofu.modulepkg \
  module.zip:archive/zip

# Alternative: standard OCI image layer (tar.gz)
tar czf module.tar.gz -C /path/to/module .
oras push ghcr.io/my-org/terraform-modules:v1.2.3 \
  --artifact-type application/vnd.terraform.module.v1+tar \
  module.tar.gz:application/vnd.oci.image.layer.v1.tar+gzip
```
