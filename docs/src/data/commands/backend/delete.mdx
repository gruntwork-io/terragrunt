## Delete State

Using this command deletes the backend state file for the current Terragrunt unit.

For example, given the following `remote_state` block:

```hcl
# terragrunt.hcl
remote_state {
  backend = "s3"
  config = {
    bucket  = "mybucket"
    key     = "path/to/my/key"
    region  = "us-east-1"
    encrypt = true

    dynamodb_table = "tf-lock"

    accesslogging_bucket_name = "mybucket-logs"
  }
}
```

Running `terragrunt backend delete` will delete the backend state file located at `path/to/my/key` in the `mybucket` bucket.

### Azure Backend

For the `azurerm` backend, `delete` removes the state blob from the Azure Storage container. Terragrunt will prompt for confirmation before deleting.

```hcl
# terragrunt.hcl
remote_state {
  backend = "azurerm"
  config = {
    storage_account_name = "myterragruntstate"
    container_name       = "tfstate"
    resource_group_name  = "terraform-rg"
    subscription_id      = "12345678-1234-1234-1234-123456789abc"
    key                  = "app/terraform.tfstate"
    use_azuread_auth     = true
  }
}
```

Running `terragrunt --experiment azure-backend backend delete` will delete the state blob at `app/terraform.tfstate` in the `tfstate` container.

Use `--non-interactive` to skip the confirmation prompt, or `--force` to delete even when blob versioning is not enabled on the storage account.
