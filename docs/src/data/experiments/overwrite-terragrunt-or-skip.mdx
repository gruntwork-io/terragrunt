A new `if_exists` mode for `generate` blocks that skips generation when a file exists without a Terragrunt signature, rather than returning an error.

### `overwrite-terragrunt-or-skip` - What it does

When `if_exists = "overwrite_terragrunt_or_skip"` is set on a `generate` block, Terragrunt checks whether the target file was previously generated by Terragrunt. If it was, the file is overwritten as normal. If it was not (e.g. a file committed to the repository by hand), generation is skipped silently and the existing file is left unchanged.

This fills the gap between `overwrite_terragrunt`, which errors when a non-Terragrunt file is present, and `skip`, which skips even files that Terragrunt itself wrote.

A common use case is a repository where a hand-written `backend.tf` coexists with a Terragrunt-managed one in different units. With this mode, Terragrunt leaves the hand-written file alone instead of failing.

### `overwrite-terragrunt-or-skip` - How to enable

```bash
terragrunt apply --experiment overwrite-terragrunt-or-skip
```

Or via environment variable:

```bash
TG_EXPERIMENT='overwrite-terragrunt-or-skip' terragrunt apply
```

### `overwrite-terragrunt-or-skip` - Criteria for stabilization

To stabilize this feature, the following need to be resolved, at a minimum:

- [ ] Gather feedback from users on the expected behavior when `disable_signature = true` is set alongside this mode.
- [ ] Add integration tests covering the case where the existing file does have a Terragrunt signature (overwrite path).
