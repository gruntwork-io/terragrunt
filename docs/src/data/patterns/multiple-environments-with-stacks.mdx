When you have several environments (dev, stage, prod) that should look the same
but differ in a few inputs, a [Stack](/features/stacks/) lets you define the
shape of an environment once and instantiate it per environment.

## The pattern

Define the units that make up an environment in a `terragrunt.stack.hcl`:

```hcl
unit "vpc" {
  source = "${get_repo_root()}/units/vpc"
  path   = "vpc"
}

unit "app" {
  source = "${get_repo_root()}/units/app"
  path   = "app"
}
```

Instantiate the stack per environment, passing environment-specific values:

```hcl
# environments/dev/terragrunt.stack.hcl
include "root" {
  path = find_in_parent_folders()
}
```

Running `terragrunt stack run apply` from an environment directory provisions all
of its units in dependency order.

## Why it works

- The stack definition is the single source of truth for an environment's shape.
- Per-environment values stay isolated, so promoting a change from dev to prod is
  a matter of running the same stack in another directory.

See [Stacks](/features/stacks/) for the full reference.
